# Easy Tracker

![Easy Tracker截图](./screenshots/easy-tracker-screenshot-zh-cn.jpeg)

[English](https://github.com/hunter-ji/obsidian-easy-tracker#easy-tracker) | 简体中文

**Easy Tracker 是一个 Obsidian 插件，让你在日记中极简追踪目标和习惯。**

在日记中直接追踪目标进展——只需插入预设模块，每天一键打卡。每天的数据独立存储在 frontmatter 中，完美兼容 Obsidian Bases，支持高级查询和分析。

## 功能简介

- 一键插入打卡模块（概览、热力图、按钮组、目标清单、笔记计数器）到任意日记。
- 每天点击按钮记录进展，数据保存到 frontmatter。
- 概览、热力图、连续天数等信息一目了然。
- 通过交互式清单追踪多个目标。
- 通过笔记计数器监控每日笔记创建数量。
- 随时编辑历史打卡记录。
- 完全兼容 Obsidian Bases 数据库查询。
- 支持中英文界面，周起始日可选。

## 使用方法

1. 在 **设置 → 社区插件** 启用 **Easy Tracker**。
2. 打开一篇日记（或任何在 frontmatter 中有 `date` 字段的笔记）。
3. 通过命令面板选择 `插入打卡组件`，一键插入所有模块。
4. 每天点击按钮完成打卡，无需额外设置。
5. 也可按需插入或删除单个模块。

## 主要模块

### 每日概览
显示当前笔记的状态、连续天数、最近缺席日期。统计数据相对于笔记日期计算，查看过去的笔记会显示历史数据。

### 年度热力图
可视化全年进展，数据聚合自库中所有包含打卡值的笔记。

**按文件夹筛选（可选）：**
你可以指定文件夹，让热力图只显示该文件夹中的笔记数据：
```markdown
```easy-tracker-year-calendar-heatmap
{
  "folder": "日记"
}
```
````

如果不指定文件夹，热力图将扫描整个库中的所有笔记。

**示例：**

  顺利完成 | 2
  额外完成 | 3

## 数据存储

所有数据存储在笔记的 frontmatter 中，具有以下优势：
- **可搜索**：使用 Obsidian 搜索或 Dataview 查询
- **可同步**：兼容任何同步方案
- **数据库就绪**：兼容 Obsidian Bases
- **可移植**：纯 YAML 格式

**frontmatter 示例：**
```yaml
---
date: 2026-03-03
tracker: 2                    # 打卡值
tracker-goals: [0, 2]         # 已完成目标的索引
---
```

## Obsidian Bases 集成

由于打卡数据存储在 frontmatter 中，你可以创建强大的数据库视图：

1. 在 Obsidian 中创建新的 Base
2. 设置来源为你的日记文件夹
3. 按 `tracker` 字段筛选（不为空）
4. 以表格形式查看所有打卡记录
5. 排序、筛选和分析你的习惯数据

## 设置选项

### 每周起始日
选择周日或周一作为一周的开始（影响热力图布局）。

### 日期字段
用于读取笔记日期的 frontmatter 字段名（默认：`date`）。

### 打卡值字段
用于存储打卡值的 frontmatter 字段名（默认：`tracker`）。

### 允许编辑打卡记录
启用后可以更新过去日期的打卡值。启用时：
- 显示当前值并高亮对应按钮
- 点击任意按钮可更改值
- 适用于纠正错误或更新历史数据

## 个性化

- **按钮标签/数值**：编辑 `easy-tracker-buttons` 代码块内的文本。
- **目标列表**：编辑 `easy-tracker-goal-checklist` 代码块来增删目标。
- **模块排列**：模块相互独立，可自由移动或删除。
- **字段名称**：在设置中自定义 frontmatter 字段名以匹配你的工作流。

## 命令列表

- `插入打卡组件`：添加所有模块（概览、热力图、按钮、目标清单）
- `插入单按钮打卡组件`：添加带单个打卡按钮的模块
- `插入日历热力图`：仅添加热力图
- `插入每日概览`：仅添加概览
- `插入目标清单`：仅添加目标清单
- `插入笔记计数器`：仅添加笔记计数器

## 使用场景

### 日常习惯追踪
追踪运动、阅读、冥想或任何日常习惯。热力图一目了然地显示你的坚持程度。

### 项目进度
使用打卡值评估每日进展（1 = 少量，2 = 良好，3 = 优秀）。回顾你的生产力模式。

### 目标管理
创建每日目标清单，完成后勾选。每天的清单相互独立。

### 历史分析
通过 Obsidian Bases 集成，你可以：
- 计算平均打卡值
- 找出最高效的日子
- 追踪目标完成率
- 导出数据进行外部分析

## 使用技巧

- **笔记独立性**：每篇日记都有自己的打卡状态和目标清单。
- **灵活日期**：插件从 frontmatter 读取日期，适用于任何日记格式。
- **编辑模式**：在设置中启用"允许编辑打卡记录"以更新过去的记录。
- **多重追踪**：可以在不同笔记中使用不同的打卡组件来追踪不同习惯。
- **文件夹筛选**：在热力图代码块中使用 `folder` 参数，在不同文件夹中追踪不同习惯。
- **笔记计数器**：笔记计数器显示的是笔记日期当天的创建数，而非总是"今天"。

无需复杂设置，插入即用，轻松追踪你的目标。
